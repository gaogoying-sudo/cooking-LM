Definition

CLM 不是单独模型，也不是单独 Agent

它是一套围绕菜谱、经验、规则、设备和输出执行组织起来的内部智能系统。

项目真正要建设的是：把口味工程师现场工作中最核心、最可复用、最贴近真实交付价值的部分，逐步拆成对象、知识、规则、流程与输出能力，而不是把“大模型会不会说”当成项目主体。

不是

不是单独模型项目

模型只是系统中的局部能力，不是数据库、规则库和流程引擎的替代品。

不是

不是先做一个会说话的 Agent

Agent 可以串联能力，但如果没有对象、知识与规则底座，只会变成入口层包装。

是

是一套可持续建设的交付系统

先围绕高频任务跑通最小闭环，再把素材逐步加工成知识资产和输出能力。

Phase Goal

第一阶段核心目标

第一阶段不是追求一上来满分，而是让系统在真实约束下交付一个可用首版。

客户在口味工程师不到场的情况下，先起出 60–70 分首版菜谱。

这意味着系统首先要承接的是“首版可起出、可试做、可反馈、可继续优化”的能力，而不是直接把所有菜一次性做到稳定高分。第一阶段更偏迁移和辅助交付，不是全面替代经验本身。

Three Scenes

三类核心场景

三类场景都重要，但不能同优先级并行推进。

场景 1

客户已有菜，目标是迁移复刻

最接近真实交付，已有目标菜、已有人工经验、已有设备执行约束，是最适合作为 Phase 1 主样板的场景。

  • 目标清楚
  • 知识可积累
  • 反馈闭环最短
场景 2

客户没有现成菜谱，但市场上存在类似菜

系统可以依托相似案例、外部资料和已有知识做参考收敛，但这个阶段更适合预埋能力而不是承诺稳定主交付。

  • 要预埋相似检索能力
  • 要预埋弱建议与补问机制
  • 不宜直接承诺强输出
场景 3

创意菜 / 新菜 / 数据库里没有参考

这是未来重要能力方向，但不是当前阶段的主要交付目标，否则会把项目过早拉向开放式生成与高不确定性场景。

  • 目前不作为主交付目标
  • 只保留长期方向说明
  • 避免稀释当前资源
Phase 1 Strategy

Phase 1 主攻策略

当前建议：主攻场景 1，兼顾场景 2 的能力预埋，不把场景 3 作为第一阶段主要交付目标。

主攻 已有菜迁移复刻，围绕“首版起出、试做反馈、继续修正”跑通最小闭环。
兼顾 预埋相似菜检索、弱建议、补问与转人工能力，为场景 2 提前铺底。
暂缓 不把创意菜 / 新菜作为当前主要承诺对象，避免模型能力叙事反客为主。
