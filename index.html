Platform Definition

我们整体在建什么

围绕原始输入、知识沉淀、规则边界、模型推理和输出执行组织起来的 Cooking LM 平台。

当前总界面只承担 5 件事：锁定项目一句话定义、说明当前阶段目标、给出两个主视角入口、承接核心专题入口、汇总当前状态与历史版本。

项目一句话定义

把原始输入加工成系统资产，再让系统在真实需求下做分级输出与正确失败。

当前阶段目标

先把高频任务、真实样板、知识沉淀和可回查产出结构搭稳。

当前主版本

平台重组版，强调“总界面 + 视角页 + 专题群 + 治理页”的分层组织。

历史版本入口

右上角 `历史版本` 按钮可直接回查旧版主入口、快照页和旧专题页。

Two Main Views

两个主视角入口

这两个视角解决不同问题：一个讲系统如何被建设出来，一个讲系统在运行时如何工作。

建设视角页

回答“我们如何把原始输入加工成系统资产”。主归属：输入分类、原始数据接入层、结构化加工层、知识沉淀层、规则形成机制、中间资产分类。

运行视角页

回答“当客户提出真实需求时，系统如何工作”。主归属：需求输入、检索层、规则与策略层、模型推理层、输出执行层、正确失败机制。

Core Topic Entry

核心专题入口

总界面不再把专题正文堆在首页，而是把它们归位到对应专题页与子任务页。

技术交付专题页

最终交付物、系统组成、核心对象、知识状态、阶段地图和投入边界。

案例与样板专题页

案例矩阵、样板案例、强建议 / 弱建议 / 转人工 / 知识建设队列。

阶段性工作产出专题页

产出总览总台、子任务入口、成果沉淀与页面关联。

进展与治理专题页

任务拆解、甘特图、里程碑、页面归位、版本治理和 README 映射。

Current Status

当前状态摘要

当前平台不是内容从零开始，而是在保留旧版本的前提下做上层重组。

已经立住的部分

总界面、建设视角、运行视角、技术交付专题、案例与样板专题、阶段性工作产出专题、进展与治理专题已经分层归位。
