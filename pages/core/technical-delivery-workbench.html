Definition

最终交付物不是单独模型，也不是单独 Agent

最终交付物是一套以公司私有语料为原材料，通过结构化抽取、知识沉淀、检索、规则、模型推理和 Agent 编排构成的内部智能系统。

这一页优先承接之前的“技术交付工作台”内容，但按新主叙事重组展示：先定义最终交付系统，再看八层架构、十大技术模块、核心对象、知识状态流转、阶段性交付地图以及模型 / 本地化 / 显卡投入边界。

Delivery System

最终交付系统由哪些部分组成

数据库负责存，模型负责理解和推理，Agent 负责把多个能力串起来完成任务，真正交付的是系统而不是单点对象。

数据库

负责存放主数据、版本和业务状态。

素材库

负责沉淀视频、纪要、图片、日志等原始证据。

知识库

负责存放规则、模板、失败模式和标杆菜谱。

向量检索库

负责相似召回和弱匹配场景支撑。

模型服务

负责局部理解、比较、排序、解释与建议生成。

Agent 工作流

负责把检索、规则、模型和输出动作串起来。

规则引擎

负责风险控制、流程收口、人工确认与下发边界。

前端工作台

负责承载审核、补问、解释和交付协同。

设备结构化输出

负责把建议结果转成设备侧可执行的数据格式。

Eight Layers

八层技术架构

八层是交付角度的结构视图，不是单纯技术栈分层。

1. 原始数据接入层

把数据库、素材、日志、专家经验和外部参考资料接进系统。

↓
2. 结构化加工层

把不同模态内容映射进统一对象体系。

↓
3. 知识沉淀层

把可复用经验提炼成规则、模式、模板和标杆菜谱。

↓
4. 检索层

从数据库、知识库和向量检索库找相关上下文。

↓
5. 规则与策略层

根据设备、风险、状态和权限做边界控制。

↓
6. 模型推理层

在允许范围内做理解、排序、解释与建议生成。

↓
7. Agent 工作流层

把多能力编排成完整任务链路。

↓
8. 输出执行层

把结果送到工作台、审核链路和设备执行侧。

Ten Modules

十大技术模块

模块清单用来回答“完整交付到底要建哪些东西”。

数据底座开发

建立主数据、版本和关系主干。

数据接入与预处理

负责接收原始素材并做基础清洗。

AI 结构化抽取服务

负责从文本、图片、视频、音频中抽取对象与标签。

知识沉淀与审核模块

负责候选知识生成、验证和版本留痕。

向量检索模块

负责相似召回和弱匹配支持。

规则引擎与策略配置

负责风险阈值、审核要求和流程收口。

在线决策服务

把检索、规则、模型和输出组织为任务链。

Agent 编排层

负责多步骤调用和任务状态推进。

前端 / 工作台

负责解释、审核、补问和交付协同。

评测与监控

负责结果评估、回流和运行状态监控。

Objects and States

核心技术对象与知识状态流转

先建对象层，再建知识类型层，再围绕任务填内容，而不是先建全量世界模型。

菜品

名称、风味、目标场景。

菜谱版本

version_id、参数包、适配版本。

工艺步骤

顺序、动作、阶段。

参数项

时长、火力、投料、速度。

调整动作 / 原因

知道改了什么，也知道为什么改。

设备上下文 / 评价结果

确保建议与设备和反馈绑定。

草稿

原始记录或初始候选，尚未进入可用知识范围。

↓
候选

AI 或人工归纳出的候选知识条目。

↓
待验证

进入人工或流程验证阶段。

↓
已验证

可以进入检索、规则和输出链路。

↓
失效

被新知识替代或不再适配当前条件。

Phases

阶段性交付地图

阶段不是平均展开，而是按先打底、再增强、再交付、再固化的顺序推进。

Phase 1

资产沉淀与知识主干

统一对象、知识结构、规则边界和最小流程。

Phase 2

AI 加工与检索增强

批量加工历史资产，补结构化抽取与相似召回。

Phase 3

策略协同与交付辅助

把建议、规则、补问和人工确认组合成稳定链路。

Phase 4

反馈闭环与能力固化

让高频稳定局部能力逐步走向规则化、本地化和专用化。

Investment Boundary

模型 / 本地化 / 显卡投入判断边界

当前不建议优先投入自建生成模型，只有当高频任务稳定、成本和延迟问题显著、且团队具备维护能力后，才逐步评估局部模型与算力投入。

现在先做

数据、知识、规则、流程和最小闭环，先把任务主线跑通。

再考虑

当在线高频任务稳定、云端成本明显上涨、延迟诉求增强时，再考虑本地化与小模型。

真要投入时优先

优先固化高频、稳定、成本敏感的局部任务，而不是先追求通用 GPT 替代品。
